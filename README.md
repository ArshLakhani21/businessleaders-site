# Business Leaders India — Site

Astro 4 static site for [businessleaders.in](https://businessleaders.in). Auto-published by the engine via `git push`. Cloudflare Pages watches `main` and rebuilds in ~30s.

**Do not commit articles or images here directly.** The engine repo (`businessleaders-engine`, private) writes to `src/content/articles/` and `public/images/articles/` and pushes. Manual edits get overwritten.

## Stack

- Astro 4.16
- Tailwind CSS 3
- Content collections with Zod schema validation
- Astro RSS + sitemap

## Local development

```bash
pnpm install
pnpm dev      # http://localhost:4321
pnpm build    # produces dist/
pnpm preview  # serves the build
```

## Cloudflare Pages settings

| Setting | Value |
|---|---|
| Framework preset | Astro |
| Build command | `pnpm install --frozen-lockfile && pnpm build` |
| Build output directory | `dist` |
| Root directory | `/` |
| Node version | `NODE_VERSION=22` (env var) |

Custom domain: `businessleaders.in`. Cloudflare provisions SSL automatically.

## Content schema

Article frontmatter must validate against [`src/content/config.ts`](src/content/config.ts). Fields:
- `title`, `slug`, `beat` (one of 9), `publishedAt`
- `author { name, slug }`
- `sources[] { id, name, url, language }`
- `image { path, alt, photographer, source, sourceUrl }`
- `metaDescription`, `tags[]`
- `wordCount`, `auditScore`, `runId`

Schema mismatch fails the Astro build, which surfaces in Cloudflare Pages.

## Routes

- `/` — homepage, latest 20 articles
- `/[beat]/` — beat archive (9 beats)
- `/[beat]/[slug]/` — article page
- `/about/`, `/contact/`, `/privacy/`
- `/rss.xml` — global RSS feed
- `/sitemap-index.xml` — auto-generated by `@astrojs/sitemap`
